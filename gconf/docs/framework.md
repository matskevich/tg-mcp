# framework: базовый цикл работы с задачей + ai

5 шагов, которые можно навесить на ЛЮБУЮ задачу:

→ РАЗГОН  
→ ПЛАНИРОВАНИЕ  
→ ПРОМПТ = ТЗ  
→ EXECUTION  
→ FEEDBACK

каждый шаг = мини-brick: своя цель, вопросы, артефакт.

⸻

## 1. РАЗГОН

цель:  
вынуть задачу из каши головы и эмоционального шума → в карту контекста.

здесь мы НЕ решаем, а **описываем поле**.

**что фиксируем**

→ контекст  
- где это вообще происходит? (работа / деньги / тело / отношения / продукт / обучение / система)  
- кто вовлечён, кроме тебя? (люди, клиенты, партнёры, процессы, ai)

→ job to be done  
- какая РАБОТА должна стать сделанной?  
- как выглядел бы мир, если бы задача была решена?

→ ограничения  
- время (дедлайн, ритм)  
- ресурсы (деньги, люди, скиллы, доступы)  
- этика / границы (что точно нельзя)

→ состояние  
- я сейчас в каком стейте? (паника, злость, апатия, азарт)  
- это состояние помогает задаче или ломает её?

**паттерн → антипаттерн**

- паттерн: «у меня проблема, помогите, хз что именно».  
- антипаттерн: «вот контекст / job / ограничения / моё состояние».

**артефакт**

- 1–2 абзаца / bullets с описанием: CONTEXT / JOB / CONSTRAINTS / STATE.  
это то, что ты потом отдашь ai и людям.

⸻

## 2. ПЛАНИРОВАНИЕ

цель:  
превратить туманный job → в траекторию: цикл и гипотезы.

**что делаем**

→ переводим в ЦИКЛ  
- ignite: почему это вообще важно двигать сейчас?  
- experiment: какой МИНИ-ШАГ проверит, что движение возможно?  
- reflect: по какому признаку пойму, что шаг удался/нет?  
- integrate: что изменю, если шаг сработает?

→ очерчиваем scope  
- что точно входит в этот цикл  
- что точно НЕ входит (не притаскиваем всё подряд)

→ собираем гипотезы  
- 2–3 вариантов, КАК можно это сдвинуть  
- не выбираем «единственно верный», а задаём поле

**паттерн → антипаттерн**

- паттерн: «сразу ищу один идеальный план / one shot».  
- антипаттерн: «у меня есть цикл + несколько гипотез, что можно протестировать».

**роль ai**

- помогает выписать варианты циклов и гипотез  
- но не решает за тебя, какой брать — выбор = твоя ответственность

**артефакт**

- схема/список: ЦИКЛ (ignite/experiment/reflect/integrate) + 2–3 гипотезы шага.

⸻

## 3. ПРОМПТ = ТЗ

цель:  
научиться относиться к промпту как к **техзаданию для агента**, а не к «поболтать с gpt».

**структура промпта как тз**

→ роль  
- кто ты просишь, чтобы модель «стала»? (аналитик, редактор, архитектор цикла, циничный консультант, коуч, наставник…)

→ входные данные  
- кратко: контекст + job + ограничения (из разгона)  
- что уже пробовали (если пробовали)

→ задача модели  
- что именно сделать с этим:  
  - разложить в цикл  
  - сгенерить 3 стратегии  
  - покритиковать план  
  - написать текст  
  - придумать эксперименты

→ критерии качества  
- как ты поймёшь, что ответ полезен?  
  - конкретика / примеры / структура / учёт ограничений  
  - формат (таблица, bullets, сценарии)

→ формат ответа  
- чётко: таблица / список / структура / псевдокод / draft-письмо / опции с плюсами/минусами

**паттерн → антипаттерн**

- паттерн: «gpt, помоги…», без контекста, цели и критериев.  
- антипаттерн: промпт как тз: роль + вход + задача + критерии + формат.

**артефакт**

- сохранённый промпт-шаблон для этой задачи/типа задач  
- дальше его можно дорабатывать и переиспользовать.

⸻

## 4. EXECUTION

цель:  
сойти со страницы → в действие. не застрять в планах и промптах.

**что делаем**

→ выбираем ОДНУ гипотезу  
- не чинить весь мир сразу  
- берём самый дешёвый по времени/стрессу шаг, который всё ещё даёт сигнал

→ запускаем эксперименты  
- делаем то, что написали с ai  
- в реальной среде: с людьми, деньгами, телом, продуктом

→ фиксируем факт действия  
- когда, что именно, в каком объёме сделал  
- не: «я про это думал», а: «я сделал N».

**паттерн → антипаттерн**

- паттерн: стопориться на доведении промпта/плана до совершенства.  
- антипаттерн: “достаточно понятно, чтобы сделать первый шаг” → делаю → потом улучшаю.

**роль ai**

- помогает разбить шаги, снять лишний страх, подготовить артефакт (текст, письмо, структуру встречи)  
- но сам эксперимент — В РЕАЛЬНОЙ ЖИЗНИ, не в чате

**артефакт**

- факт-лог: что именно сделал (дата, шаг, контекст)  
- иногда → внешний артефакт (отправленное письмо, пост, прототип, разговор).

⸻

## 5. FEEDBACK

цель:  
не просрать опыт. превратить действие → в информацию и следующую итерацию.

**3 слоя обратной связи**

→ от РЕАЛЬНОСТИ  
- что ответило окружение?  
  - люди  
  - деньги  
  - метрики  
  - тело (сон, энергия, уровень стресса)

→ от AI  
- что скажет модель, если показать ей:  
  - «я сделал вот это → вот результат»  
  - попросить:  
    - разобрать, что сработало/нет  
    - предложить следующую итерацию  
    - указать на слепые зоны в решении

→ от ЛЮДЕЙ / СИСТЕМЫ  
- фидбек от людей, кому это касается (партнёры, клиенты, gconf)  
- системный фидбек: как это вписывается/ломает другие процессы

**паттерн → антипаттерн**

- паттерн: «получилось/не получилось» → эмоция → забыли → следующая попытка с нуля.  
- антипаттерн: «получилось/не получилось» → выписали выводы → скорректировали промпт/план → новая итерация.

**ключевые вопросы**

- что сработало НЕ так, как я думал?  
- что я хочу сохранить из этого способа действия?  
- что я изменю в промпте/плане/цикле в следующем заходе?  
- какой артефакт я могу оставить для себя/других?

**артефакт**

- короткий пост-мортем / заметка (3–5 bullet’ов)  
- обновлённый промпт или схема  
- фикс: «следующая итерация → вот такая».

⸻

## TL;DR: ОДНА ФОРМУЛА

любой кейс мы учим прогонять так:

1) РАЗГОН — назвать поле и job.  
2) ПЛАНИРОВАНИЕ — перевести в цикл + гипотезы.  
3) ПРОМПТ = ТЗ — оформить задачу для ai как для агента.  
4) EXECUTION — сделать минимальный шаг в реальности.  
5) FEEDBACK — собрать обратную связь и обновить цикл.

это и есть “framework для любой задачи”, к которому потом уже цепляются конкретные bricks (звонки, практики, лумы, бот-ритуалы).
